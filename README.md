# REST API "Перевал" (Pereval Project)

## Описание проекта
Проект представляет собой полнофункциональное REST API для системы модерации данных о горных перевалах. Разработано в рамках учебной стажировки на Python с использованием фреймворка **FastAPI**.

Система позволяет:
- Добавлять новые заявки на перевалы с геоданными, уровнями сложности и изображениями.
- Просматривать список всех перевалов с фильтрацией.
- Получать детальную информацию по конкретному перевалу.
- Обновлять статусы модерации заявок.
- Удалять записи.

Особенность реализации: работа с **PostgreSQL 16**, гибкая настройка через переменные окружения и поддержка базы данных `pereval` (специфическая конфигурация без пароля учтена в коде).

## Технологический стек
- Язык: Python 3.10+
- Фреймворк: FastAPI, Uvicorn (ASGI сервер)
- Валидация данных: Pydantic
- База данных: PostgreSQL 16
- Документация: Swagger UI (встроена автоматически)
- Тестирование: pytest, httpx
- Контроль версий: Git

## Пошаговая инструкция по запуску

### Шаг 1. Клонирование репозитория
Открой терминал и выполни команду для скачивания проекта:
git clone https://github.com/Gustavo666322/pereval-fastapi
Затем перейди в папку проекта:
cd pereval-fastapi

### Шаг 2. Создание виртуального окружения
Создай изолированное окружение для проекта:
python -m venv venv

Активируй его:
- Для Windows:
venv\Scripts\activate
- Для Linux / macOS:
source venv/bin/activate

### Шаг 3. Установка зависимостей
Установи все необходимые библиотеки из файла requirements.txt:
pip install -r requirements.txt

Убедись, что в файле требований присутствуют: fastapi, uvicorn, sqlalchemy (или asyncpg), psycopg2-binary, pydantic-settings, pytest, httpx.

### Шаг 4. Настройка переменных окружения
Создай в корне проекта файл с именем .env и запиши в него следующие настройки:

FSTR_DB_HOST=localhost
FSTR_DB_PORT=5432
FSTR_LOGIN=postgres
FSTR_PASS=postgre7676
DB_NAME=pereval
APP_SECRET_KEY=your-secret-key-change-this
DEBUG=true

Примечание: Логика подключения к базе данных pereval уже настроена в коде с учетом требований задания.

### Шаг 5. Запуск сервера разработки
Запусти приложение используя сервер Uvicorn. Если твой главный файл называется main.py:
uvicorn main:app --reload

Если главный файл называется иначе (например, app.py), замени main на имя своего файла:
uvicorn app:app --reload

После запуска сервер будет доступен по адресу:
http://127.0.0.1:8000

## Документация API (Swagger)
Так как проект написан на FastAPI, интерактивная документация генерируется автоматически и не требует дополнительной настройки.

После запуска сервера открой в браузере:
http://127.0.0.1:8000/docs

Там ты сможешь:
- Просмотреть все доступные эндпоинты.
- Изучить структуру запросов и ответов (схемы Pydantic).
- Протестировать API прямо из браузера, нажав кнопку "Try it out".

### Список основных эндпоинтов:
POST /submitData/ — Создание новой заявки на перевал
GET /submitData/ — Получение списка всех перевалов (с фильтрацией)
GET /submitData/{id} — Получение данных конкретного перевала по ID
PATCH /submitData/{id} — Частичное обновление данных (например, статуса модерации)
DELETE /submitData/{id} — Удаление записи

## Покрытие тестами
Проект покрыт автоматическими тестами, которые проверяют корректность работы с базой данных и логику API.

Для запуска тестов выполни команду в терминале (предварительно убедившись, что виртуальное окружение активировано):
pytest

Тесты проверяют:
1. Успешное создание объекта через POST-запрос.
2. Корректное получение созданных данных через GET-запрос.
3. Работу валидации при отправке некорректных данных.
